Перегляд логів наживо через USB

Коли модуль не виходить онлайн, дивіться повне завантаження через USB на платі, що вже працює:

python3 provision.py --monitor

Це не прошиває й не переконфігуровує нічого. Команда транслює лог прошивки — лінк Ethernet, плати входів, що виявлено, отриманий IP, підключення MQTT, HA discovery — з помилками червоним і попередженнями жовтим. Натисніть Ctrl-C, щоб зупинити.

Щоб слухати стабільний польовий модуль без перезапуску, додайте --monitor-passive:

python3 provision.py --monitor --monitor-passive

Виведення модуля з експлуатації

# Повне виведення: прибрати з Home Assistant ТА стерти плату.
python3 provision.py --uninstall-all

# ...або окремі кроки:
python3 provision.py --uninstall-firmware   # стерти файли застосунку (MicroPython лишається)
python3 provision.py --erase-flash          # стерти ВСЕ, включно з MicroPython
python3 provision.py --uninstall-ha         # прибрати з Home Assistant

--erase-flash стирає всю flash-пам’ять, лишаючи «голий» RP2040 у BOOTSEL. Команда спершу підтверджує (незворотно). Щоб повторно використати плату, перезапустіть provision.py.
