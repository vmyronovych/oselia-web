## Поширені повідомлення

| Повідомлення | Ймовірна причина / виправлення |
|--------------|--------------------------------|
| `No RP2040-ETH detected over USB` | Кабель не під’єднано або не той порт — спробуйте `--port`. Якщо плата зникла *після* провіженінгу, відновіть через **BOOTSEL** (тримайте BOOT під час під’єднання), потім перезапустіть. |
| `No MicroPython detected` / невідповідність версії | Прийміть пропозицію майстра прошити закріплену збірку (потрібен інтернет, або передайте `--mpy-uf2 PATH`). |
| `BOOTSEL drive (RPI-RP2) didn't appear` | Плата не ввійшла в BOOTSEL; зробіть BOOT+RESET вручну й повторіть. |
| `could not enter raw repl` | Вотчдог робочого модуля бореться з `mpremote`. Перепідключіть (або натисніть RESET) і перезапустіть. |
| `cannot reach <ip>:<port>` | Неправильний IP/порт брокера, брокер вимкнено, або ноутбук в іншій мережі. |
| `broker refused: bad username or password` | Виправте облікові дані й перезапустіть. |
| `FAIL (ethernet)` | Кабель Ethernet / IP брокера недоступні **з мережі шлюзу**. |
| `FAIL (mqtt)` | Брокер доступний, але логін/сесія не вдалися — перевірте IP/порт і облікові дані. |
| `FAIL (mcp)` | Плата входів не відповідає — перевірте проводку I²C і що кількість плат відповідає встановленим чипам. |
| `zeroconf package missing` | Прийміть пропозицію майстра встановити його, або відмовтеся й введіть IP брокера вручну. |

## Перегляд логів наживо через USB

Коли модуль не виходить онлайн, дивіться повне завантаження через USB на платі, що
вже працює:

```bash
python3 provision.py --monitor
```

Це **не прошиває й не переконфігуровує** нічого. Команда транслює лог прошивки — лінк
Ethernet, плати входів, що виявлено, отриманий IP, підключення MQTT, HA discovery —
з помилками червоним і попередженнями жовтим. Натисніть Ctrl-C, щоб зупинити.

Щоб слухати стабільний польовий модуль **без перезапуску**, додайте
`--monitor-passive`:

```bash
python3 provision.py --monitor --monitor-passive
```

## Виведення модуля з експлуатації

```bash
# Повне виведення: прибрати з Home Assistant ТА стерти плату.
python3 provision.py --uninstall-all

# ...або окремі кроки:
python3 provision.py --uninstall-firmware   # стерти файли застосунку (MicroPython лишається)
python3 provision.py --erase-flash          # стерти ВСЕ, включно з MicroPython
python3 provision.py --uninstall-ha         # прибрати з Home Assistant
```

`--erase-flash` стирає всю flash-пам’ять, лишаючи «голий» RP2040 у BOOTSEL. Команда
спершу підтверджує (незворотно). Щоб повторно використати плату, перезапустіть
`provision.py`.
