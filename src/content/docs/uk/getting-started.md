Цей посібник проведе новий **OSELIA Hearth (DI16-G)** від коробки до роботи в Home
Assistant. Програмувати нічого не треба: ви під’єднуєте USB-кабель, запускаєте одну
команду й відповідаєте на кілька запитань. Решту робить майстер.

> ⚠️ **Ніколи не під’єднуйте USB-C, коли живлення 24 В увімкнене.** Плата не може
> живитися від USB і 24 В одночасно. **Вимкніть живлення 24 В перед під’єднанням
> USB** і вмикайте його лише після від’єднання USB. Для провіженінгу USB само по
> собі живить шлюз нормально.

## Перш ніж почати

Вам потрібно:

- **Hearth**, підключений (входи 24 В + **під’єднаний Ethernet**), з **вимкненим
  живленням 24 В** наразі. MicroPython *не обов’язково* мати заздалегідь — майстер
  перевірить інтерпретатор і запропонує прошити закріплену збірку, якщо її немає.
- **USB-кабель** від шлюзу до ноутбука. USB живить шлюз під час провіженінгу;
  Ethernet лишається під’єднаним, щоб дотягтися до брокера.
- **Home Assistant**, що вже працює в локальній мережі, з MQTT-брокером, у **тій
  самій мережі**, куди під’єднано Ethernet шлюзу.
- Ваші **логін / пароль брокера**, якщо брокер вимагає авторизацію.

## Одноразове налаштування ноутбука

Встановіть Python 3, потім інструмент, яким майстер спілкується з платою:

```bash
pip install mpremote
# опційно: краща серійна діагностика; обов’язково для --monitor-passive у Windows
pip install pyserial
```

`zeroconf` (для автопошуку брокера) спершу не обов’язковий — майстер запропонує
встановити його під час першого запуску.

## Провіженінг шлюзу

1. **Переконайтеся, що живлення 24 В вимкнене**, потім під’єднайте шлюз до ноутбука
   через USB. Лишіть **Ethernet** під’єднаним; живлення 24 В лишається вимкненим на
   весь цей крок.

2. З теки `provisioning/` репозиторію прошивки запустіть:

   ```bash
   python3 provision.py
   ```

3. Відповідайте на запити. Майстер **знаходить MQTT-брокери в мережі автоматично**
   (mDNS, потім швидке сканування порту 1883):

   | Запит | Що робити |
   |-------|-----------|
   | `Which MQTT broker number` | Лише якщо знайдено кілька — введіть номер потрібного. |
   | `Broker IP or hostname` | Лише якщо нічого не знайдено — введіть адресу брокера. |
   | `Broker username` | Введіть, або лишіть **порожнім** для анонімних брокерів. |
   | `Broker password` | Введіть (прихований), або лишіть порожнім. |

   Кількість плат входів **визначається автоматично** — шлюз сканує шину I²C під час
   завантаження, тож кількості у вас не питають.

4. Майстер перевіряє доступність брокера **перш ніж щось записувати**, записує
   конфігурацію, копіює прошивку, перезапускає плату й чекає, доки вона повідомить
   **online через брокер**.

5. За **`PASS: device … is online`** — **від’єднайте USB, потім увімкніть живлення
   24 В.** Шлюз перезавантажується на польовому живленні, перепідключається, і входи
   оживають.

Готово — шлюз тепер пристрій у Home Assistant. Далі встановіть
[інтеграцію Home Assistant](/uk/docs/ha-integration), щоб він з’явився під OSELIA з
OTA-оновленнями, потім [назвіть вимикачі й створіть автоматизації](/uk/docs/automations).

## Додавання розширювача Ember

Модулям [OSELIA Ember (DI-16X)](/uk/products/ember-di16x) **не потрібен власний
провіженінг** — шлюз виявляє їх на своїй шині I²C:

1. Виставте адресні перемички модуля (A0–A2) на наступну вільну адресу (`0x21`,
   `0x22`, …).
2. Підключіть його на спільну польову шину поруч із Hearth.
3. **Перезавантажте живлення шлюзу.** Він пересканує шину під час завантаження, і
   нові входи з’являться як `board<N>_input<M>` у Home Assistant.

## Перенесення шлюзу в іншу мережу

Усе специфічне для майданчика (IP брокера, логін MQTT, DHCP/статика) живе в
маленькому `site.json` на платі, і шлюз **не** переналаштовує себе сам. Щоб
перенести: перезапустіть `python3 provision.py` на новому майданчику — він
автоматично знайде новий брокер і запише нову конфігурацію. **Перепрошивка не
потрібна** — переїзд це суто переконфігурація.

## Переконфігурація пізніше

Запускайте `python3 provision.py` повторно на тому самому модулі будь-коли. Він
читає наявні налаштування, пропонує їх за замовчуванням і дає змінити лише брокер,
замінити облікові дані чи додати плату. Повторний запуск безпечний.
