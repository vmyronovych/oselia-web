Інтеграція **OSELIA Hearth** для Home Assistant робить так, що шлюз з’являється під
**власною інтеграцією OSELIA** (а не загальною MQTT-інтеграцією) з рідною сутністю
оновлення прошивки **update** для OTA. Встановлюється один раз на інстанс Home
Assistant.

## Встановлення через HACS (рекомендовано)

1. У Home Assistant відкрийте **HACS → ⋮ (вгорі праворуч) → Custom repositories**.
2. Репозиторій: `https://github.com/vmyronovych/oselia-hearth-di16g-ha`,
   категорія **Integration**.
3. Встановіть **OSELIA Hearth**, потім **перезапустіть Home Assistant**.
4. **Settings → Devices & Services → Add integration → OSELIA Hearth** і вкажіть ваш
   MQTT-брокер.

Після цього оновлення приходять у HACS автоматично з кожним релізом.

> Майстер провіженінгу може налаштувати інтеграцію за вас (`--ha-setup`), але сам
> *компонент* інтеграції спершу має бути встановлений через HACS, як вище.

## Встановлення вручну

Скопіюйте `custom_components/oselia/` у `config/custom_components/` вашого Home
Assistant, перезапустіть Home Assistant і додайте інтеграцію як у кроці 4 вище.

## Як це працює

- Інтеграція відкриває **власне MQTT-з’єднання** до того самого брокера, що й шлюзи —
  вона **не** залежить від MQTT-інтеграції Home Assistant. Один запис конфігурації на
  брокер.
- Шлюзи виявляються з їхніх retained-топіків (`hearth/<id>/status`,
  `…/diag/state`). Кожен стає одним пристроєм Home Assistant під OSELIA.

## Що ви отримуєте для кожного шлюзу

- **Події** — одна сутність `event` на вхід, з типами `single` / `double` / `long`.
- **Тригери пристрою** — на кожен вхід `button_short_press` / `button_double_press` /
  `button_long_press`, тож вони з’являються у виборі **Device → Trigger**.
- **Діагностика** — аптайм, вільна пам’ять, температура RP2040, перепідключення,
  втрачені події, плати онлайн + їхні адреси, останній вхід, IP, лінк Ethernet.
- **Керування** — кнопки Restart / Identify; живі числові параметри таймінгу;
  вибір рівня логів. Зміни діють миттєво й зберігаються після перезавантаження.
- **Оновлення прошивки** — рідна картка OTA; `latest_version` з GitHub-фіду релізів.
  OTA перевірено апаратно від початку до кінця (завантаження → перевірка →
  застосування → підтвердження, з авто-відкатом за збою).

## Стійкість

Інтеграція **безпечна при старті** (брокер, недоступний на старті HA, ніколи не
зриває налаштування), **автоматично перепідключається** з повторною підпискою,
слідкує за доступністю брокера, надає завжди-доступний сенсор **«Broker
connection»** і **безпечна до перезавантаження**.
